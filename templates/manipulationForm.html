? Epsilon defines a range around the student's initial budget within which the algorithm checks for fairness. For example, if epsilon is set to 0.5 and a student's initial budget is 5, then the algorithm will consider budgets ranging from 4.5 to 6.5 for this particular student.

? Delta is used to update the courses prices in ACEEI algorithm. For example, if delta is set to 0.5 and the price of course x is 5 so the update will be x = 5 + 5 * 0.5 = 7.5.

? Eta is used to adjust the true preferences of the selected student. For example, if eta = 2, each preference for every course will be modified so that one preference is multiplied by 2, while another is divided by 2.

? Beta is used to define the range of the distribution from which the students' budgets are drawn. For example, if Beta = 4, the budgets will be randomly selected within the range from 1 + (beta/4) to 1 + (beta*3)/4.

? EF-TB status is used to define the envy condition. EF-TB ensures that there is no envy between a student with a higher budget and a student with a lower budget. FEF-TB CONTESTED also addresses envy when course prices are zero.

? Criteria for Profitable Manipulation determines whether only the budgets of the students are sampled, or if the preferences of other students are also included.
