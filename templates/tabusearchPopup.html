TABU SEARCH

Purpose

The TABU SEARCH algorithm aims to find a fair allocation of courses for students. It adjusts course prices based on demand to reach a fair distribution.

Algorithm Overview

The main idea of the algorithm is to adjust course prices based on demand:

  • Increase Prices: For courses with excess demand (where too many students want the course).
  • Decrease Prices: For courses with a lack of demand (where too few students want the course).

Iteration Process

The algorithm operates in a loop and stops when the clearing error is zero. Hereâ€™s how it works:

  1. Add the current prices to a history of similar prices. Similar prices are those that result in the same allocation and the same excess demand.
  2. Evaluate two types of neighbors:
    • Gradient Neighbors: These are obtained by multiplying the current prices by a delta and adding the error vector.
    • Individual Price Adjustments:
      • Increase prices for courses with excess demand in such a way that only one student can no longer afford the course.
      • Set prices for courses with a lack of demand to zero.
  3. For each neighbor, calculate the allocation and determine the error.
  4. Select the neighbor that results in the minimum error and update the current prices.

Termination

The algorithm terminates when it finds a price vector that results in a zero error. At this point, it returns the final prices and the resulting course distribution.
